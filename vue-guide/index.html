Introduction

Overview

With this guide you'll be able to learn how to install Vue and tailwindCSS with Vite, what Vue is and the basics of Vue.

What is Vue.js?

Vue (pronounced /vjuː/, like view) is a progressive framework for building user interfaces. Unlike other monolithic frameworks, Vue is designed from the ground up to be incrementally adoptable. The core library is focused on the view layer only, and is easy to pick up and integrate with other libraries or existing projects. On the other hand, Vue is also perfectly capable of powering sophisticated Single-Page Applications when used in combination with modern tooling and supporting libraries. (source)

What is Vite?

Vite (French word for "fast", pronounced /vit/) is a build tool that aims to provide a faster and leaner development experience for modern web projects. It consists of two major parts:

  • A dev server that provides rich feature enhancements over native ES modules, for example extremely fast Hot Module Replacement (HMR).
  • A build command that bundles your code with Rollup, pre-configured to output highly optimized static assets for production.

Vite is opinionated and comes with sensible defaults out of the box, but is also highly extensible via its Plugin API and JavaScript API with full typing support.

You can learn more about the rationale behind Vite in the Why Vite page. (source)

What is tailwindCSS?

TailwindCSS is a utility-first CSS framework packed with classes like flex, pt-4, text-center and rotate-90 that can be composed to build any design, directly in your markup. (source)

Installation And Setup

Install Vue with Vite

We can easily scaffold a Vue project with Vite with the following command, this will create a project folder named vue-app:

npm init vite@latest vue-app

Or if you're using yarn:

yarn create vite vue-app

You should see the following, use your arrow keys as stated in the cli to select Vue and press enter:

? Select a framework: › - Use arrow-keys. Return to submit. vanilla ❯ vue react preact lit-element svelte

Again, use your arrow keys to select vue and press enter:

✔ Select a framework: › vue ? Select a variant: › - Use arrow-keys. Return to submit. ❯ vue vue-ts

And with that we've scaffoled our project! Now we'll install the dependencies, run the following:

cd vue-app && npm install

Or with yarn:

cd vue-app && yarn install

And we're done installing Vue! Next we'll install tailwindCSS.

Install and setup tailwindCSS

Install tailwindCSS and its dependencies with the following command:

npm install -D tailwindcss@latest postcss@latest autoprefixer@latest

Or with yarn:

yarn add -D tailwindcss@latest postcss@latest autoprefixer@latest

Next, we'll create the config files:

npx tailwindcss init -p

Or with yarn:

yarn tailwindcss init -p

We'll edit our tailwind.config.js file and replace the purge field with the following:

Understanding The Boilerplate

Directory structure

Your directory structure should look like this:

vue-app/ ├── node_modules ├── public │   └── favicon.ico ├── src │   ├── assets │   │   └── index.css │   ├── components │   ├── App.vue │   └── main.js ├── index.html ├── package-lock.json ├── package.json ├── postcss.config.js ├── tailwind.config.js └── vite.config.js

Understanding the folders

Lets walkthrough the folders in our vue-app directory:

  • node_modules - This is where all your dependencies are installed.
  • public - This is where you'll store your assets that aren't referenced in your source code or must retain the exact same file name (without hashing). Assets in this directory will be served at root path / during dev, and copied to the root of the dist directory as-is.
  • src - This is where your entry file and other source code (e.g. components and pages) will be stored.
  • src/assets - This is where you'll store assets/css that you'll import into your source code, file names will get hashed when built like index.cc6b0949.css.
  • src/components - This is where you'll store your .vue components. Components are reusable instances and they accept the same options as a root instance, such as data, computed, watch, methods, and lifecycle hooks.

Understanding the files

Lets walkthrough the files in our vue-app directory:

  • public/favicon.ico - This is the default favicon icon Vite's Vue template comes with.
  • src/assets/index.css - This is the css file we created in the setup section. Tailwind will inject its css as defined by the @tailwind directive.
  • src/App.vue - This is the root component, basically the starting point for your app.
  • src/main.js - This is where you'll import Vue, mount the root component to the dom, add Vue router, vuex etc.
    In there we are importing the createApp function from vue, our css and the root component. Then we use the function to mount the root component on the div with id app defined in index.html.
  • index.html - You can see a div with an id app this is where vue will mount the app. Edit it to include your apps title, description, anything you'd do in a html file.
  • package.json, package-lock.json - You know what these are! they're of course used to manage your dependencies. You'll have yarn.lock instead of package-lock.json if you're using yarn.
  • tailwind.config.js, postcss.config.js - They are used to configure tailwindCSS and postCSS respectively. Learn more about tailwind's config file here.
  • vite.config.js - Vite's config file. You can add plugins, pass config for rollup etc. here. Learn more about Vite's config file here.

Code you see below applies to App.vue.

Single File Components

Introduction

Vue Single File Components (aka *.vue files, abbreviated as SFC) is a special file format that allows us to encapsulate the template, logic, and styling of a Vue component in a single file. The App.vue file you have in your src directory is a SFC! Here's an example SFC:

As we can see, Vue SFC is a natural extension of the classic trio of HTML, CSS and JavaScript. Each *.vue file consists of three types of top-level language blocks: <template>, <script>, and  <style>:

  • The <script> section is a standard JavaScript module. It should export a Vue component definition as its default export.
  • The <template> section defines the component's template.
  • The <style> section defines CSS associated with the component.
(source)

How it works

Vue SFC is a framework-specific file format and must be pre-compiled by @vue/compiler-sfc into standard JavaScript and CSS. A compiled SFC is a standard JavaScript (ES) module - which means with proper build setup you can import an SFC like a module:

(source)

Why SFC

While SFCs require a build step, there are numerous #benefits in return:

  • Author modularized components using familiar HTML, CSS and JavaScript syntax.
  • Pre-compiled templates
  • Component-scoped CSS
  • More ergonomic syntax when working with Composition API
  • More compile-time optimizations by cross-analyzing template and script
  • IDE support with auto-completion and type-checking for template expressions
  • Out-of-the-box Hot-Module Replacement (HMR) support
(source)
